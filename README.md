🥋 CT Ferroviário - Dashboard & Management System
Bem-vindo ao repositório do ecossistema digital do CT Ferroviário de Judô. O que nasceu como uma Landing Page institucional evoluiu para uma plataforma completa de gestão administrativa para dojos, focada em controle de atletas, graduações e saúde financeira.

Este projeto utiliza o que há de mais moderno no ecossistema React para entregar uma experiência de usuário (UX) rápida, responsiva e robusta.

🚀 Evolução do Projeto
Originalmente concebido em 2025 como uma Landing Page para divulgação, o projeto foi totalmente reformulado em 2026 para integrar-se a um Backend robusto em Spring Boot, transformando-se em uma ferramenta de gestão ativa.

✨ Features Principais (Painel Administrativo)
🔍 Filtros Inteligentes Multi-Critério: Sistema de busca em tempo real que permite cruzar dados de nome, status de atividade (Ativo/Inativo), gênero e situação financeira.

🌙 Dark Mode Nativo: Interface adaptável com suporte a modo escuro para melhor conforto visual em diferentes ambientes de treino.

📊 Gestão de Atletas & Graduações: Controle total sobre a evolução técnica (faixas) e horários (turnos) dos alunos.

💰 Controle Financeiro Integrado: Monitoramento de mensalidades com sinalização visual de pendências e baixa de pagamento com um clique.

📄 Geração de Documentos: Integração com API para download de Fichas Técnicas em PDF e geração de relatórios administrativos em formato CSV (Excel).

📱 Design Mobile-First: Dashboard totalmente operacional em tablets e smartphones, facilitando o uso pelo Sensei diretamente no tatame.

🛠️ Stack Tecnológica
Framework: Next.js 14+ (App Router & Client Components)

Linguagem: TypeScript (Tipagem estrita para maior segurança)

Estilização: Tailwind CSS (Design moderno e utilitário)

Comunicação: Fetch API para consumo de API REST assíncrona.

Deploy: Vercel

🔗 Integração com o Backend
Este Front-end é a face visível de um ecossistema desacoplado. Ele consome a API REST do CT Ferroviário, desenvolvida em Java/Spring Boot, que gerencia a persistência de dados em PostgreSQL e a lógica de automação de mensalidades.

Nota para Recrutadores: A separação em repositórios distintos reflete uma arquitetura de microserviços escalável, permitindo que a API sirva tanto esta interface Web quanto futuros aplicativos mobile.
